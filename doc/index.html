TypeScriptって何？

  • AltJS !
    • JavaScriptにコンパイルできる言語
  • C#を作ったアンダースさんが作ったよ
  • JavaScriptのかゆいところに手が届くよ
    • 静的型付け/クラスベースのオブジェクト指向/名前空間など
  • ほぼ同じ構造のリーダブルなJSが出来上がるよ
  • Apache License 2.0

その他のAltJS (1)

  • CoffeeScript
    • Rails標準
    • でも静的型付けではない
    • 読みやすいJSを出力
  • Haxe
    • コンパイル後のJSが最適化される
    • Flash,PHP,Java,C#とかにもコンパイルできるらしい

その他のAltJS (2)

  • Dart
    • Google社製
    • コンパイルしてネイティブ化もできる
    • JavaScriptの置き換えを狙っている
    • 最適化
  • JSX
    • DeNA社製
    • 最適化

サンプル作ってみました

メリット

  • Visual Studio で使ってみて
    • インテリセンス
    • デバッグ
  • 既存JSライブラリと連携が容易
    • NuGetで宣言ファイルとライブラリ本体を入手可
    • 有名どころの宣言ファイルはたいてい公開されている(DefinitelyTyped)

デメリット

  • referenceだるい
  • htmlファイルにjsパスの追加忘れがち
  • 公開されている宣言ファイル0.9対応(主にgenerics)されてない場合あり
  • 公開されてなかったり、自作のJSライブラリの場合は宣言ファイルも自作
  • IDEとの連携がまだ弱い

ところで

現代のJSライブラリの充実ぶりすごい

私の知識がPrototype.jsが流行りだした頃で止まってたので…

サンプルで使ってみた

  • testerm
    • テストランナー(更新自動検知!)
  • jasmine
    • ユニットテスト(Spec形式)
  • knockout.js
    • MVVMパターンフレームワーク
  • underscore.js
    • コレクション処理

コーディングの様子

  • testermを起動させておく
  • ブラウザでテストランナーを表示する
  • jasmineでSpecを書く
  • 実装
  • ビルド
  • テストが自動実行される(゜д゜)ｳﾏｰ

使ってないけど気になる

  • QUnit ・・・ ユニットテスト(XUnit形式)
  • Backbone.js ・・・ MVCフレームワーク
  • grunt ・・・ ビルドツール
  • bower ・・・ パッケージマネージャ
  • RequireJS ・・・ モジュールの動的ロード

まとめ

  • ちょうどいい塩梅の静的型付けでサクサク書ける
  • 既存ライブラリを利用して短いコードでいろいろできる
  • 今後も目が離せない！！(Ver1.0待望)
